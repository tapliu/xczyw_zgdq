# 信长之野望-战国夺旗 v0.0.4

基于日本战国历史的回合制策略棋盘游戏。玩家扮演一方大名，通过招募武将、部署军队、征战沙场，夺取敌阵旗帜。

## 特性

- **100 名战国武将** — 统、武、智、政四维属性，S+/S/A/B/C/D 评级
- **14 个势力 + 多势力系统** — 27 名武将可同时隶属多个势力（如羽柴秀吉=织田家/丰臣家），同势力友军邻接时获得兵力加成，敌对势力邻接时削弱
- **势力关系系统** — 每个势力有君主、同盟、敌对关系，直接影响战场阵型策略
- **类型加成系统** — 全能/文臣/武将/特才四种类型，开局自动按比例增幅属性
- **三选一招募** — 每回合从随机武将中选择 1 名加入麾下
- **旗本系统** — 指定大名身份武将作为旗本，阵亡后掉落旗印可被友军拾取；棋盘以旗帜图标标记，己方金色/敌方绿色
- **武将编辑器** — 编辑现有武将属性，支持保存/恢复
- **自建武将** — 20 个头像位，支持新建/编辑/保存自定义武将，类型受属性比例约束
- **自动战斗** — 近战/远程/侧击/夹击/争夺，全自动结算，含详细命中记录与均伤统计
- **结算画面** — 显示 MVP（含势力标签、造成/受伤害均伤）、击杀/近战/远程伤害 TOP3（含均伤）、MVP 造成/受伤害 TOP3（含均伤）
- **列队前进** — 前排推进时后排自动补位，保持阵型
- **地形系统** — 支持不同战场地形（天王山、长篠等）
- **武将头像** — WebP 格式武将画像，含 20 张自定义头像（头像生成器）
- **响应式棋盘** — 棋盘区域占页面 47.5% 宽度，自适应缩放

## 势力与关系系统

### 势力构成

游戏包含 14 个势力，每个势力有君主（大名）、同盟、敌对关系：

| 势力 | 君主 | 主要敌对 |
|---|---|---|
| 织田家 | 织田信长 | 武田、上杉、毛利等 |
| 丰臣家 | 丰臣秀吉 | 德川、毛利等 |
| 德川家 | 德川家康 | 武田、上杉等 |
| 武田家 | 武田信玄 | 织田、德川、上杉等 |
| 上杉家 | 上杉谦信 | 武田、北条、织田等 |
| 北条家 | 北条氏康 | 上杉、武田等 |
| 毛利家 | 毛利元就 | 织田、大友等 |
| 岛津家 | 岛津义弘 | 大友、龙造寺等 |
| 伊达家 | 伊达政宗 | — |
| 长宗我部家 | 长宗我部元亲 | 织田、毛利等 |
| 龙造寺家 | 龙造寺隆信 | 岛津、大友等 |
| 大友家 | 大友宗麟 | 岛津、龙造寺、毛利等 |
| 三好家 | 三好长庆 | 织田等 |
| 群雄 | — | 中立 |

### 多势力武将

27 名武将隶属多个势力，例如：
- **丰臣秀吉** = 织田家（主）+ 丰臣家
- **真田昌幸** = 武田家（主）+ 上杉家 + 德川家
- **黑田官兵卫** = 织田家（主）+ 丰臣家

武将的`主势力`（primary）用于界面的势力标签颜色和君主判定，`全部势力`（all）用于战场势力加成计算。

### 势力对战斗力的影响（faction_bonus）

每个单位的战力计算流程中，势力加成（faction_bonus）扫描该单位 8 个邻接格子：

| 条件 | 效果 |
|---|---|
| 邻接有**同势力友军** | `faction_bonus += 0.05`（若单位含`群雄`则 +0.03） |
| 邻接有**敌对势力敌军** | `faction_bonus -= 0.05` |

势力敌对关系由`relations.json`中的 enemies 数组定义，敌对是双向判定。

### 君主加成（Lord Bonus）

若单位是所属势力的君主（大名），且不属于`群雄`，则额外获得：
```
power *= (1 + 同势力友军数 × 0.05)
```
即每个同势力友军再增幅 5% 战力。

### 数据文件结构

`server/src/data/relations.json` 包含三部分：

**char_factions** — 武将→势力映射：
```json
"丰臣秀吉": { "primary": "织田家", "all": ["织田家", "丰臣家"] }
```

**factions** — 势力关系：
```json
"织田家": { "lord": "织田信长", "allies": ["德川家康"], "enemies": ["武田家", "上杉家"] }
```

**char_relations** — 武将间关系（叙事元数据）：
```json
"织田信长": [{ "type": "主君-家臣", "target": "丰臣秀吉" }]
```

## 属性详解

每个武将拥有四维属性，在游戏中各有独立作用：

### 统率（leadership）

- **直接提升战力**：`战力 += 统率 × 旗本倍率 × 0.05`
- **增益友军**：邻接友军获得该单位统率 × 0.05 的战力加成
- **决定兵力上限**：`最大兵力 = min(MAX_TROOPS, 统率 × 100)`
- **AI 分配兵力**：AI 部署时 `兵力 = 武力 × 60 + 统率 × 40`

### 武力（martial）

- **主要伤害属性**：`战力 += 武力 × 2 × 旗本倍率`
- 是近战伤害的核心来源

### 智力（intelligence）

- **远程伤害系数**：`远程伤害 = 兵力 × (攻击方战力/总战力) × (智力/100) × 0.25`
- **削弱敌方**：位于敌方正面两行的单位，其战力被敌方智力 × 0.03 衰减

### 政治（politics）

- **减少战损**：`战损系数 = 1 - 政治/240`（政治越高，自军受伤越少）
- **战后恢复**：`恢复率 = min(自身政治+邻接友军政治总和 × 0.001, 0.6)`
- **收入加成**：`收入倍率 = clamp(全体武将政治总和 × 0.001, 0.2, 2.0)`

### 旗本倍率

被指定为旗本的武将获得 1.1× 属性倍率（作用于统率和武力的战力贡献）。

## 武将类型

游戏开局时，类型自动对武将属性进行增幅（不影响原始属性显示）：

| 类型 | 效果 |
|---|---|
| **全能** | 四维全部 × 1.05 |
| **文臣** | 智力、政治 × 1.07 |
| **武将** | 统率、武力 × 1.07 |
| **特才** | 最高单项属性 × 1.10 |

## 评级系统

评级根据武将四维总和分布自动计算：

| 评级 | 条件 |
|---|---|
| **S+** | 任意属性 = 100 |
| **S** | 剩余武将中前 1/11（按总属性排序） |
| **A** | 接下来 3/11 |
| **B** | 接下来 5/11 |
| **C** | 接下来 8/11 |
| **D** | 末尾 |

评级影响 AI 的部署优先级和观战席招募倾向。

## 战斗公式摘要

### 单位战力
```
power = 武力 × 2 × 旗本倍率
power ×= (1 + 兵力/30000)
power += 统率 × 旗本倍率 × 0.05
power += 邻接友军统率 × 0.05
power ×= faction_bonus
power ×= 君主加成（若适用）
power ×= 包围衰减（三面包围 × 0.9）
power -= 敌方正面智力 × 0.03（若在敌方锥形范围）
```

### 近战伤害
```
伤害 = 攻击方兵力 × (攻击方战力/双方总战力) × 0.8
自损 = 攻击方兵力 × (防守方战力/双方总战力) × 0.8 × (1 - 防守方政治/240)
```

### 远程伤害
```
伤害 = 攻击方兵力 × (攻击方战力/总战力) × (智力/100) × 0.25
无自损
```

### 侧击/夹击
- **侧击**：若攻击方格子的斜后方有友军，该友军参战
- **夹击**：若目标两侧（左/右）均有友军，触发夹击
- 侧击/夹击时，攻击方战力获得加成

## 游戏规则

- **棋盘**: 8×8 网格，玩家控制下半区（4-7行），AI 控制上半区（0-3行）
- **身份系统**: 武将分为大名（64人）与非大名（36人），仅大名可被选为旗本
- **卡牌招募**: 每回合从 3 张随机武将中选择 1 张加入麾下（三选一）
- **旗本抽取**: 第一轮抽牌完成后，双方从抽牌堆中抽取 3 名大名作为旗本候补
- **部署**: 每回合在本方区域部署最多 5 个单位，分配兵力
- **战斗**: 双方部署完成后自动结算，支持近战、远程、侧击、夹击
- **胜利条件**:
  1. 达阵 — 任意单位进入敌方底线
  2. 全歼 — 消灭所有敌方单位
  3. 旗本溃散 3 次 — 敌方旗本武将阵亡 3 次

## 快速开始

```bash
# 安装依赖
pip install -r server/requirements.txt

# 启动服务器
python -m server.src.main
# 或: uvicorn server.src.main:app --reload

# 浏览器访问
# http://127.0.0.1:8000
```
